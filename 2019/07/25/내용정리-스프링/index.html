Spring이란?

  • 엔터프라이즈 어플리케이션에서 필요로 하는 기능을 제공하는 프레임워크
  • 동적인 웹 사이트를 개발하기 위한 여러 가지 서비스를 제공

스프링 특징

  1. 경량 컨테이너
    : 자바 객체의 라이프 사이클을 관리하며 스프링으로부터 필요한 객체를 얻는다.
  2. IoC(Inversion of Control)
    : 컨트롤의 제어권이 사용자가 아닌 프레임워크에 있다.
  3. DI(Dependency Injection)
    : 각 계층이나 서비스 간 의존성을 프레임워크가 연결시켜준다.
  4. AOP(Aspect-Oriented Programming)
    : 트랜젝션이나 로깅과 같이 여러 모듈에서 공통으로 사용하는 기능을 분리하여 관리

1. Spring Container

Spring Container는 bean객체의 생명주기를 관리한다.

  1. BeanFactory
    : 기본적인 의존성 주입을 지원하는 가장 간단한 컨테이너
  • 팩토리 디자인 패턴 구현
  • 빈을 생성하고 분배하는 역할
  • getBean() 메소드가 호출될 때까지 Bean 생성을 미룸(lazy loading)
  1. ApplicationContext
    : 애플리케이션 프레임워크를 제공해 BeanFactory 개념 위에 구현된 컨테이너
  • BeanFactory와 유사하지만 더 많은 기능을 제공
  • Singleton Bean을 미리 로딩하여, 필요할 때 즉시 사용될 수 있도록 보장. -> 빈 생성되기를 기다릴 필요가 없다.

2. DI(Dependency Injection)

  • People 객체가 Hobby 객체를 참조하고 있어 의존성을 갖게 된다.
  • Hobby 객체를 변경하면 의존한 People 객체도 변경된다.
  • Maven : 의존관계에 있는 라이브러리를 알아서 다운받고 배포까지 해준다.

DI(의존성 주입)

  • 객체가 아닌 프레임워크에 의해 객체 의존성이 주입된다.
  • 프레임 워크에 의해 동적으로 의존성이 주입돼 여러 객체간 결합이 줄어든다.
  • Container가 bean 객체를 생성하고 종속성을 주입한다.

Bean 객체 생성

  • Spring Container에 bean 등록
  • id : Spring Container에서 생성된 객체 구분하는데 사용되는 식별 값
  • name : id와 마찬가지로 식별값인데 여러 개 설정 가능
  • class : 생성할 bean 객체의 패키지 포함한 클래스명
  • id는 xml 구성요소이기 때문에 동일한 id 값을 갖는 bean 태그를 설정할 수 없음.

의존성 주입 방법

  1. 생성자를 통한 의존성 주입
  • value : primitive 또는 String 타입의 값 설정
  • ref : 이미 등록된 bean의 id를 설정해 해당하는 객체 주입
  • index : 인자의 순서를 지정하고 생략시엔 순서대로 매핑
  • type : 인자의 타입 지정

ref 태그는 레퍼런스를 의미해, board 클래스 생성자에 식별 값이 title인 빈 객체 전달.
생성자가 전달 받는 값이 primitive or String type 이라면, ref 대신 value로 값을 지정.

  1. setter를 통해 의존성 주입
  • name : 속성 이름
  • value : primitive 또는 String 타입의 값 설정
  • ref : 이미 등록된 bean의 id를 설정해 해당하는 객체 주입

  • 종속성 감소
  • 재사용성 증가
  • 코드를 읽기 쉬워짐
