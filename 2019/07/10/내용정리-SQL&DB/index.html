내용정리 - SQL&DB편

Database란

기록 정보를 모아둔 것을 데이터 베이스라 한다.

  1. 관계형 데이터베이스
    열과 행을 갖는 2차원 테이블로, 데이터를 key와 value라는 연결 고리로 연결하여 데이터를 조회하고 가공하는 데이터 베이스.
  2. 계층형 데이터베이스
    컴퓨터 저장 장치에 주로 사용하는 방식으로, 부모 - 자식 관계로 데이터를 표현하는 데이터 베이스.
  3. 객체지향 데이터베이스
    객체와 객체 식별자, 속성과 메서드, 클래스, 클래스 계층 및 상속 등 객체 지향 데이터 모델을 지원하는 데이터베이스
  4. XML 데이터베이스
    XML 표준 문서 구조를 계층형 트리 형태로 저장하거나 관리하는 데이터베이스
    #### 데이터베이스 구성요소
  5. 개체
    데이터 베이스에서 표현하고자 하는 유형, 무형의 객체로서 서로 구별 되는 것.
    ex) 고객, 학생, 과목, 학과, …
  6. 속성
    개체의 특성이나 상태.
    ex) 학생(학번, 이름, 학년, 주소, …)
  7. 관계
    개체와 개체 혹은 개체와 속성간의 관계
    #### 테이블 상세 구조
  8. 행 : 특정 개체에 대한 모든 데이터를 나타내는 단일 행
  9. 기본 키(primary key) : 유일하게 데이터를 구분하는 열로, 데이터를 식별하는 ‘식별자’. 중복값과 null 값을 가질 수 없다.
  10. 고유 키(unique key) : 행에서 유일한 값을 갖는 데이터 값으로 구성 된 열. 키본키가 아닌 데이터 값이 존재한다.
  11. 외래 키(foreign key) : 테이블 간에 서로 연결 관계를 정의하는 외래 키. 테이블 구성 열이자 다른 테이블과 연결을 위한 키.
    ## SQL이란? > 관계형 데이터베이스를 조작하기 위해 만들어진 표준 언어.
    ### SQL 특징 1) 사용하기 쉬움. 2) 절자가 없는 비절차적 언어 3) 관계형 데이터베이스의 데이터 조작 가능

SQL 명령어 종류

종류 기능 명령어
DML
(Data Manipulation Language)		 데이터 베이스의 논리적, 물리적 구조를 정의 SELECT
INSERT
UPDATE
DELETE
DDL
(Data Definition Language)		 데이터 베이스에 접근하여 데이터를 검색, 생긴, 삽입, 삭제 CREATE
ALTER
DROP
RENAME
TRUNCATE
DCL
(Data Control Language)		 데이터베이스에 접근하고 객체들을 사용하도록 권한을 주고 받음. GRANT
REMOVE
TCL
(Transaction Control Language)		 데이터 베이스의 내용이 정확성과 안전성을 유지하도록 제어 COMMIT
ROLLBACK
SAVEPOINT

#### 1. SELECT문

EMP테이블의 전체 데이터를 출력하는 쿼리문.

emp 테이블에서 직원들의 empid, first_name, last_name를 출력하는 쿼리문.

ORDER BY

ORDER BY를 사용하여 행 정렬 순서를 지정할 수 있다.
ASC는 오름차순 정렬(default), DESC는 내림차순 정렬이다.

emp 테이블에서 직원들의 empid, first_name, last_name를 내림차순 정렬해 출력하는 쿼리문.

DISTINCT

DISTINCT 명령어는 중복된 행을 제거한다.

emp 테이블에서 jobid를 중복제거해 출력하는 쿼리문.

AS

명명된 열 이름 외에 임의로 바꿔 쓰고자 할때 사용하는 별칭이다.

emp테이블에서 empid는 사원번호, first_name는 이름 그리고 last_name은 성으로 출력하는 쿼리문.

WHERE

조건에 맞는 데이터를 조회할 때 사용하는 것이 WHERE절이다. WHERE절에서는 연산자를 사용하여 데이터 값을 조작할 수 있다.

  • 비교 연산자
    비교 연산자는 프로그래밍 언어와 비슷하다. 프로그래밍 언어에서 같지 않다를 !=로 표현하지만 SQL문은 <>로도 표현 가능하다.

  • SQL 연산자

    • BETWEEN 연산자
      BETWEEN 연산자는 두 값의 범위에 해당하는 행을 출력할 때 사용.

    • IN 연산자
      조회하고자 하는 데이터 값이 여러 개일 때 사용.
      =연산자와 유사하지만 =는 조회 조건으로 데이터 값을 하나만 지정 가능.

    • LIKE 연산자
      조회 조건 값이 명확하지 않을 때 사용.
      LIKE 연산자는 %와 _같은 기호 연산자와 함께 사용. %는 ‘모든 문자’라는 의미이고 _는 ‘한 글자’라는 의미.

    • IS NULL 연산자
      데이터 값이 null인 경우를 조회하고자 할 때 사용.

  • 논리 연산자
    AND, OR, NOT으로 SQL 문의 조건을 계소 추가해야 할 때 필수 연산자.


#### 2. DML 데이터를 직접 삽입, 생긴, 삭제하는 명령어.

INSERT
테이블에 새로운 행을 삽입할 때 사용.

UPDATE
기존의 데이터 값을 다른 데이터 값으로 변경할 때 사용

INSERT, UPDATE 명령어로 데이터를 삽입하거나 갱신할 때는 입력되는 데이터의 순서, 데이터 타입과 제약 조건(NOT NULL, UNIQUE 등)에 주의해야 한다.

DELETE
테이블의 데이터를 삭제할 때 사용.


#### 3. DDL 테이블과 관련 열을 생성하고 변경하고 삭제하는 명령어

CREATE
새로운 테이블을 생성할 때 사용.

ALTER
이미 생성한 테이블에 열을 추가, 변경, 삭제하여 테이블의 구조를 변경

  • 열 추가

  • 열 수정

  • 열 삭제

TRUNCATE
테이블의 데이터를 모두 삭제하고 사용하던 기억 공간도 해제되지만 테이블 구조는 삭제되지 않는다.

DROP
테이블에 있는 모든 바료와 그와 연관된 모든 인덱스를 삭제하고 사용하고 있던 저장공간을 돌려준다.
