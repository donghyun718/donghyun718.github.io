Options / Data

1. data

object type

function type

new Vue{
    data : function() {
        return {
            a : 1
        }
    }
}

  • data를 Object와 function로 정의하는 것의 차이가 무엇일까?
    단일 컴포넌트라면 Object로 사용해도 괜찮지만, 부모 컴포넌트가 있거나 자식 컴포넌트가 있다면 Object 타입의 data는 같은 객체를 공유하기 때문에 한 컴포넌트에서 값을 변경하면 다 반영된다.
    이를 해결하기 위해, function으로 새로운 객체 데이터를 반환한다.

2. component

<div id="example">
    <simple-counter></simple-counter>
    <my-component></my-component>
</div>

<script>
// 등록
var Child = {
    template: '<div>사용자 정의 컴포넌트 입니다!</div>'
}

Vue.component('simple-counter', {
    template: '<button v-on:click="counter += 1">{{ counter }}</button>',
    // 데이터는 기술적으로 함수이므로 Vue는 따지지 않지만
    // 각 컴포넌트 인스턴스에 대해 같은 객체 참조를 반환합니다.
    data: function () {
        return {
            counter: 0
        }
    }
})

new Vue({
    el: '#example',
    components: {
    // <my-component> 는 상위 템플릿에서만 사용할 수 있습니다.
        'my-component': Child
    }
})
</script>

  • global/local component 등록

3. prop

Vue.component('props-demo-simple', {
    props: ['size', 'myMessage']
})

// 유효성 검사를 포함한 객체 구문
Vue.component('props-demo-advanced', {
    props: {
        // 타입 체크만 합니다.
        height: Number,
        // 타입 체크와 유효성 검사를 합니다.
        age: {
            type: Number,
            default: 0,
            required: true,
            validator: function (value) {
                return value >= 0
            }
        }
    }
})

  • 부모 컴포넌트의 데이터를 받을 수 있는 속성

4. computed

var vm = new Vue({
    data: { a: 1 },
    computed: {
        // get만 가능합니다.
        aDouble: function () {
            return this.a * 2
        },
        // get과 set 입니다.
        aPlus: {
            get: function () {
                return this.a + 1
            },
            set: function (v) {
                this.a = v - 1
            }
        }
    }
})
vm.aPlus   // => 2
vm.aPlus = 3
vm.a       // => 2
vm.aDouble // => 4
  • Vue 인스턴스에 추가되는 계산된 속성
  • 모든 getter & setter 자동으로 this 컨텍스트를 Vue 인스턴스에 바인딩
  • 계산된 속성은 캐시되며 의존하고 있는 반응형 속성이 변경되는 경우 다시 평가.
  • 특정한 의존성이 인스턴스의 범위를 벗어나는 경우(반응형이 아닌 경우)에 계산된 속성은 갱신되지 않음. 반응형 속성을 갖고 있지 않아 이를 수정하는 경우, DOM 갱신 발생하지 않음.
