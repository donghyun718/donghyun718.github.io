Vue

Options / Data

1. data

object type

function type 

new Vue{
    data : function() {
        return {
            a : 1
        }
    }
}

  • data를 Object와 function로 정의하는 것의 차이가 무엇일까?
    단일 컴포넌트라면 Object로 사용해도 괜찮지만, 부모 컴포넌트가 있거나 자식 컴포넌트가 있다면 Object 타입의 data는 같은 객체를 공유하기 때문에 한 컴포넌트에서 값을 변경하면 다 반영된다.
    이를 해결하기 위해, function으로 새로운 객체 데이터를 반환한다.

4. computed

var vm = new Vue({
    data: { a: 1 },
    computed: {
        // get만 가능합니다.
        aDouble: function () {
            return this.a * 2
        },
        // get과 set 입니다.
        aPlus: {
            get: function () {
                return this.a + 1
            },
            set: function (v) {
                this.a = v - 1
            }
        }
    }
})
vm.aPlus   // => 2
vm.aPlus = 3
vm.a       // => 2
vm.aDouble // => 4
  • Vue 인스턴스에 추가되는 계산된 속성
  • 모든 getter & setter 자동으로 this 컨텍스트를 Vue 인스턴스에 바인딩
  • 계산된 속성은 캐시되며 의존하고 있는 반응형 속성이 변경되는 경우 다시 평가.
  • 특정한 의존성이 인스턴스의 범위를 벗어나는 경우(반응형이 아닌 경우)에 계산된 속성은 갱신되지 않음. 반응형 속성을 갖고 있지 않아 이를 수정하는 경우, DOM 갱신 발생하지 않음.

4. methods

  • 인스턴스에 추가할 메소드
  • 인스턴스를 통해 직접 접근하거나 디렉티브 표현식에 사용 
var vm = new Vue({
    data: { a: 1 },
    methods: {
        plus: function () {
            this.a++
        }
    }
})
vm.plus()
vm.a // 2

5. watch

var vm = new Vue({
    data: {
        a: 1,
        b: 2,
        c: 3,
        d: 4,
        e: {
            f: {
                g: 5
            }
        }
    },
    watch: {
        a: function (val, oldVal) {
        console.log('new: %s, old: %s', val, oldVal)
        },
        // 문자열 메소드 이름
        b: 'someMethod',
        // 깊은 감시자
        c: {
            handler: function (val, oldVal) { /* ... */ },
            deep: true
        },
        // 콜백은 관찰이 시작된 직후 호출됩니다.
        d: {
            handler: function (val, oldVal) { /* ... */ },
            immediate: true
        },
        e: [
            function handle1 (val, oldVal) { /* ... */ },
            function handle2 (val, oldVal) { /* ... */ }
        ],
        // watch vm.e.f's value: {g: 5}
        'e.f': function (val, oldVal) { /* ... */ }
    }
})
vm.a = 2 // => new: 2, old: 1
  • key : 표현식, val : callback
  • Vue 인스턴스는 생성시 객체의 각 항목에 대해 $watch()를 호출
  • watch에 화살표 함수 사용은 안됨.

computed vs watch vs method

<div id="demo">
    <input type="text" v-model="firstName">
    <input type="text" v-model="lastName">

    <button v-on:click="Full">이름출력</button>
    <br/>
    <input type="text" v-model="fullName">
</div>

methods 

new Vue({
    el: '#demo',
    data: {
        firstName: 'Foo',
        lastName: 'Bar',
        fullName: 'Foo Bar'
    },
    methods:{
        Full : function(){
        this.fullName = (this.firstName + this.lastName);
        }
    }
});
  • methods는 key가 메소드 이름
  • 호출되면 함수 수행

computed 

new Vue({
    computed :{
        fullName: function () {
            return this.firstName + ' ' + this.lastName
        }
    }
});
  • computed는 명시를 안하면 기본적으로 get임
  • computed는 key가 메소드 이름
  • 함수에 종속된 변수의 값(예시. firstName, lastName)이 바뀌면 수행

watch 

new Vue({
    watch: {
        firstName: function (val) {
            this.fullName = val + ' ' + this.lastName
        },
        lastName: function (val) {
            this.fullName = this.firstName + ' ' + val
        }
    }
});
  • key 값으로 지정한 변수의 값이 바뀌면 수행

computed vs watch 

<div id="app">
    <h1>{{ num }}</h1><br>
    {{ calculated }}
    <button @click="count --">-</button>
</div>

<script>
new Vue({
    el: "#app",
    data: {
        num: 3,
    },
    computed: {
        calculated: function () {
            if(this.num === 2) {
                alert('num is 2')
            }
        }
    },
    watch: {
        count: function (newVal) {
            if(newVal === 0) {
                alert('num is 0')
                this.num = 3
            }
        }
    }
})
</script>
  • \{\{calculated}}를 호출하는 부분이 없으면 num 값이 변경돼도 함수가 수행되지 않아 출력해줘야 한다.
