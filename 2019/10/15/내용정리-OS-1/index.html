운영체제 개요

1. 운영체제란

  • 하드웨어 바로 윗단에 설치되는 소프트웨어
  • 사용자 및 다른 모든 소프트웨어와 하드웨어를 연결하는 소프트웨어 계층
  • 좁은 의미 : 메모리에 상주하는 운영체제, 커널
  • 넓은 의미 : 커널 뿐 아니라 각종 시스템을 위한 유틸리티들을 광범위하게 포함

2. 운영체제 기능

  1. 컴퓨터 시스템 내의 자원을 효율적으로 관리
  2. 컴퓨터 시스템을 편리하게 사용할 수 있는 환경 제공
  3. 사용자와 운영체제 자신을 보호

3. 운영체제 분류

  1. 동기 작업 지원 여부
  • single tasking
  • multi tasking : 여러 프로그램이 자원 공유
    • 시분할 시스템
    • 다중 프로그래밍 시스템
    • 다중처리기 시스템
  1. 다중 사용자의 동시 지원 여부
  • 단일 사용자용 운영체제
  • 다중 사용자용 운영체제
    • 이메일 서버, 웹서버에 이용되는 운영체제
  1. 작업 처리하는 방식
  • 일괄 처리 : 작업 요청의 일정량을 모아서 한번에 처리
  • 시분할 : 일정한 시간 단위로 분할해 처리
  1. 실시간 운영체제
    : 정해진 시간 안에 어떠한 일을 반드시 종료됨이 보장되어야 하는 시스템
  • 경성 실시간 시스템 : 주어진 시간을 지키지 못할 경우 위함한 결과를 초래
    ex) 로켓, 원자로 제어 시스템 등
  • 연성 실시간 시스템 : 멀티미디어 스트리밍 시스템과 같이 데이터가 시간을 맞추어야 올바른 기능을 수행

4. 운영체제의 자원관리

  • CPU 스케줄링 : 매 시점 어떤 프로세스에게 CPU를 할당해 작업을 처리할 지 결정
    • FIFO
    • Round Robin
    • Priority
  • 주소를 통해 메모리 할당
    • 고정 분할 방식 : 물리적 메모리를 몇 개의 영구적인 분할
    • 가변 분할 방식 : 매 시점 프로그램 크기에 맞게 메모리 분할
    • 가상 메모리 기법 : 물리적 메모리와는 독립적으로 자신만의 가상메모리를 가져 물리메모리와 매핑
  • 인터럽트
    • 인터럽트 요청하는 장치와 발생 상황에 따라 다양한 종류
    • 장치마다 컨트롤러가 있어 인터럽트를 발생시켜 CPU에 알림
    • 운영체제 내에 정의된 인터럽트 처리 루틴을 찾아가 수행 후 직전 상태 복구
